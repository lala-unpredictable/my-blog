CSS的层叠机制

由于稍微复杂的样式表可能会存在两条甚至多条规则同时选择一个元素的情况，CSS通过一种叫做层叠机制来处理这种冲突。

层叠机制的原理就是为规则赋予不同的重要程度，重要性级别从高到低如下所示：

  1. 标注为！important的用户样式；
  2. 标注为！important的作者样式；
  3. 作者样式；
  4. 用户样式；
  5. 浏览器（或用户代码）的默认样式；

在此基础上，规则再按【选择符的特殊性】排序。特殊性高的选择符会覆盖特殊性低的选择符。如果两条规则的特殊性相等，则后定义的规则优先。

选择符的特殊性

为了量化规则的特殊性，每种选择符都对应着一个数值。这样，一条规则的特殊性就表示为每个选择符的累加数值。但是并非是我们熟悉的十进制加法，而是基于位置的累加。选择符的分类可以参考我的上一篇博客选择符的分类与使用。

任何选择符的特殊性都对应于如下4个级别，即a、b、c、d：

  • 行内样式，a为1；
  • b等于ID选择符的数目；
  • c等于类选择符、伪类选择符及属性选择符的数目；
  • d等于类型选择符和伪元素选择符的数目；

【简单来说】，如果样式被写在了元素的style属性里，那么这些样式的特殊性就最高。

然后，通过ID属性应用的规则，其特殊性高于未通过ID属性应用的规则。

同理，通过类选择符应用的规则，其特殊性高于只通过类型选择符应用的规则。

最后，如果两条规则拥有相等的特殊性，则优先应用后定义的规则。

CSS的继承机制

继承与层叠两个概念看起来有点类似，但实际上它们有着本质上的不同。

继承的概念还算比较好理解，有些属性，像颜色或字体大小，会被应用他们的元素的后代所继承。有了继承机制，可以避免给一个元素的所有后代重复应用相同的样式。
